NUOVO RULE AGGIUNTO PER PROTEGGERE FILE HTML:

# Don't rewrite .html files (NUOVO 14/03)
RewriteCond %{REQUEST_FILENAME} -f
RewriteRule \.html$ - [L]

# Map clean URLs → .html files
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteCond %{REQUEST_URI} !\.html$
RewriteCond %{DOCUMENT_ROOT}%{REQUEST_FILENAME}\.html -f
RewriteRule ^(.*)/?$ $1.html [L,QSA]

# Fallback: SPA entry point for non-matched routes
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^ index.html [QSA,L]

ROOT CAUSE ANALYSIS:
━━━━━━━━━━━━━━━━

1. .htaccess mancava di protezione per file .html
   → Rewrite rule globale ^(.*)$ index.html sovra-scriveva tutto
   → Anche /zes.html era reindirizzato a index.html

2. Production HTML conteneva: <script type="module" src="/src/index.tsx">
   → Browser cerca di caricarie: /src/index.tsx
   → Server invia HTML instead di JavaScript
   → Browser error: "MIME type is text/html, not text/javascript"
   → React non poteva bootstrappare

3. <div id="root"></div> mai riceveva React mount
   → Pagina rimane vuota

SOLUZIONE IMPLEMENTATA (14 Marzo 2026):
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  └─ Convertito: Pure static HTML con content embedded

STEP 2: Aggiunti protezione .htaccess
  └─ Nuovo rule: Proteggere file .html da rewrite
     ├─ RewriteCond %{REQUEST_FILENAME} -f
     └─ RewriteRule \.html$ - [L]
